## 二、封面图统一尺寸(竖图/横图宽度不一致)
- 现象:武侯祠(竖图)与文殊院(横图)景点封面看起来宽度/边距不同。经查两者均为单张封面、`cover_images` 为空,走同一渲染分支。
- 根因:`Utils/CoverImageView.swift` 用 `.frame(height:)` + `.frame(maxWidth:.infinity)` 两段式,`scaledToFill` 先按固定高把竖图撑成窄条,后续 maxWidth 不再拉满整行宽 → 竖图封面比横图窄。
- 修复:改为 `Color.clear` 占位定「满宽 × 固定高」框,图片以 `.overlay` + `scaledToFill` + `clipped` 裁剪填充。任意高宽比源图均渲染为同一尺寸;组件全局共用,列表缩略图/子区域页封面一并规整。
- 验证:`xcodebuild`(iphonesimulator)BUILD SUCCEEDED。

## 六、行前清单 v1 种子（`087_seed_checklist_v1.sql`）

依据两份撰写稿（通用 / 成都）核对后台 `checklist_items` 设计并生成种子。

### 设计核对结论
- 字段基本匹配（type/entry-universal-city、target_nationalities、target_cities、group_title、title_en、why_important、how_to_complete、external_links、priority、display_tags、sort_order 等）。
- 三处不一致,与用户确认处理:
  - **无 `reminder_days_before` 列**且 App 不消费 per-item 提醒 → 忽略,走全局 `checklist_settings.reminder_days`。
  - **priority `critical` 不在枚举**(required/recommended/optional)且 App 暂不渲染 → `critical→required` + 给这些条目 `display_tags=['key']`(预留 Key 角标)。
  - **why/how 文档为中文** → 先存中文草稿(HTML,App 用 HTMLContentView 渲染),后续翻译覆盖。
- App 行为:entry/universal 段标题由 type 固定为 Entry Requirements / Essential Prep(忽略 group_title);city 段标题取城市英文名;排序仅看 `sort_order`;entry 的 `target_nationalities` 留空=对所有外国人显示。

### SQL 内容
- 停用 008 旧演示清单 8 条(`cl_vpn/cl_alipay/cl_esim/...`,含与新合规口径冲突的 VPN 条)。
- 写入 22 条 v1:入境 3(entry)+ 通用 14(支付/网络/必装App/预订/健康)+ 成都 5(city,`target_cities={chengdu}`)。
- 幂等:`ON CONFLICT (id) DO UPDATE`;why/how 用 Postgres 美元引用 `$h$…$h$` 避免中文引号转义。
- 校验:行数 22(3+14+5)、`$h$` 44 对、critical 8 条带 key、成都 5 条、外链 2 条(签证检测器/Alipay)、撇号已转义。
- **未对远程库执行**;上线方式:`supabase db push` 或 SQL Editor 粘贴。
