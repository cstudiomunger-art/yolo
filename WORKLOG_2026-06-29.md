## 一、数据库 · `095_payment_flow_cms.sql`

新增 5 张 CMS 表（RLS 公开只读 + admin 增删改，仿 063/093）：

| 表 | 用途 |
|---|---|
| `payment_countries` | q1 国家 chip + 注册短信裁决（sms_tone / reg_method / sms_advice） |
| `payment_cash_rules` | q3 行程 + 现金裁决（amount_md） |
| `payment_flow_steps` | 各节点有序步骤（tool / screenshot / fail_reasons jsonb / stability_tier） |
| `payment_rescue_rungs` | 救援阶梯（applies: always / wx_only） |
| `payment_node_texts` | 各屏文案（slot: h1 / intro / callout + tone） |

扩展：

- `payment_merchant_phrases` → `speakable`、`audio_url`
- `payment_card_networks` → `note_zh` / `note_en`（绑卡策略提示）

种子数据从原型 `seed.py` 转录：11 国、4 行程、18 步骤、5 救援级、9 商家话术、屏幕文案、补充 `payment_articles`。

## 三、iOS Views · 按 node_key 分屏导航

重构 **`PaymentHelperViews.swift`**：

```
home → prep:  q1 → q2 → q3 → plan → install → register → bind → verify → use → card
       china: china → bind → verify → use → card
       rescue: 救援阶梯(CMS) → 商家话术(TTS + 预录音频)
```

- `PaymentNodeHeader` — 读 `node_texts`
- `PaymentStepsList` — CMS 步骤 + Markdown + 截图 + fail_reasons
- register/bind — alipay/wechat 分段 Picker

- verify — `volatile` 步骤单独标注「以平台实时为准」，不进 rescue
- rescue — `applies=wx_only` 在无微信卡时隐藏
