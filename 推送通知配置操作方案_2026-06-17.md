# 客服消息推送通知 · 操作方案（你手动执行的部分）

> 目标:客服在工作台回复后,用户 App **即使在后台/关闭**也能收到推送通知。
> Bundle ID:`com.chengduyuliu.yolohappy`　Supabase 项目:`edwvrriuwzaaqznklrgi`
> 代码侧(`device_tokens` 表 / `notify-user` 函数 / 设备 token 上传 / web 回复触发)**已完成**;本文是**你需要在 Apple 与 Supabase 后台手动做的配置**。

---

## 总览(5 步,按顺序)
1. Apple Developer:给 App ID 开 Push + 建 APNs 密钥(.p8)
2. Xcode:加 Push Notifications 权限
3. Supabase:设函数密钥 + 部署 `notify-user` + 确认迁移 066
4. 真机:装 App、授权通知、上传 token
5. 联调验证 + 排错

> ⚠️ 需要:**付费 Apple 开发者账号** + 一台**真机**(模拟器收不到 APNs 真实推送)。

---

## 第 1 步 · Apple Developer:开 Push + 建 APNs 密钥

### 1.1 确认 App ID 开了 Push Notifications
1. 打开 https://developer.apple.com/account → **Certificates, Identifiers & Profiles → Identifiers**。
2. 找到 `com.chengduyuliu.yolohappy` → 点进去。
3. **Capabilities** 列表里勾选 **Push Notifications**(若已勾忽略)→ 保存。

### 1.2 创建 APNs Auth Key（.p8，整个团队共用一个即可）
1. 左侧 **Keys** → 点 **+**(新建)。
2. 名称随便填(如 `YOLO APNs`)→ 勾选 **Apple Push Notifications service (APNs)** → Continue → Register。
3. **下载 `AuthKey_XXXXXXXXXX.p8`**(⚠️ 只能下载一次,妥善保存)。
4. 记下两个值:
   - **Key ID**:就是文件名里那 10 位(也显示在 Key 详情页)。
   - **Team ID**:右上角账号信息里的 10 位 Team ID(Membership 页能看到)。

## 第 2 步 · Xcode:加 Push Notifications 权限

> 这是目前**最关键的缺口**——没有它 App 拿不到 device token,整条链路是断的。

1. Xcode 打开 `YOLO.xcodeproj` → 选 **Target「YOLO」→ Signing & Capabilities**。
2. 左上 **+ Capability** → 双击 **Push Notifications**。
3. 确认下方出现 Push Notifications 区块,且 `YOLO.entitlements` 里多了 `aps-environment`(Xcode 自动加)。
4. 确认 **Automatically manage signing** 勾着,Team 选你的开发者账号(Xcode 会自动更新带 Push 的描述文件)。
5. （可选)如果以后要"App 被杀死也能后台静默更新",再到 Signing & Capabilities 加 **Background Modes → Remote notifications**;只做横幅提醒可不加。

## 第 3 步 · Supabase:函数密钥 + 部署 + 迁移

### 3.1 设置 Edge Function 密钥
Supabase Dashboard → **Project Settings → Edge Functions → Secrets**(或 Functions → Manage secrets),新增以下 6 个:

| 名称 | 值 |
|---|---|
| `APNS_KEY_P8` | 打开 `.p8` 文件,把**全文**粘进来(含 `-----BEGIN PRIVATE KEY-----` 和 `-----END PRIVATE KEY-----` 两行) |
| `APNS_KEY_ID` | 第 1.2 步的 Key ID(10 位) |
| `APNS_TEAM_ID` | 第 1.2 步的 Team ID(10 位) |
| `APNS_BUNDLE_ID` | `com.chengduyuliu.yolohappy` |
| `APNS_HOST` | 开发/TestFlight 用 `api.sandbox.push.apple.com`;App Store 正式版用 `api.push.apple.com` |

> 命令行设密钥也行:
> ```bash
> supabase secrets set APNS_KEY_ID=xxxxxxxxxx APNS_TEAM_ID=xxxxxxxxxx \
>   APNS_BUNDLE_ID=com.chengduyuliu.yolohappy APNS_HOST=api.sandbox.push.apple.com
> supabase secrets set APNS_KEY_P8="$(cat AuthKey_XXXXXXXXXX.p8)"
> ```

### 3.2 部署 notify-user 函数
```bash
cd /Users/vesperal/Desktop/YOLO
supabase functions deploy notify-user
```

### 3.3 确认迁移 066(device_tokens 表)已应用
SQL Editor 跑一下确认表在:
```sql
select to_regclass('public.device_tokens');   -- 非空即已建
```
没有的话执行 `supabase/migrations/066_support_chat_commercial.sql`(或 `supabase db push`)。

## 第 4 步 · 真机:装 App、授权、上传 token

1. 用真机跑 App(Xcode 直接 Run 到你的 iPhone,或发 TestFlight)。
2. App 引导里**允许通知权限**(已有 onboarding 申请;若当时拒了,去 iOS 设置 → YOLO → 通知 手动打开)。
3. **触发 token 上传**:目前代码在「打开 Genius Bar」时注册并上传 device token——所以登录后**进一次首页「真人客服」**即可。
   - (开发侧待办:把注册时机提前到登录/启动,届时无需此步。)
4. 验证 token 已入库:SQL Editor
   ```sql
   select user_id, platform, updated_at from device_tokens order by updated_at desc limit 5;
   ```
   应看到你这个用户的一行。

## 第 5 步 · 联调验证

1. **A = 真机 App**(普通用户登录,已进过一次 Genius Bar)。
2. **B = 客服工作台**(https://yolosupport.cstudiomunger.workers.dev,客服账号登录、点「上线」)。
3. A 找该客服发条消息 → B 收到 → **B 回复**。
4. 把 **A 退到后台或锁屏** → 几秒内应收到推送横幅;点开 → 进入 App。
   - (点通知直接跳到客服会话:开发侧待补 deep-link 处理;现在点开是进 App 首页。)

## 第 6 步 · 排错（最常见的几个）

| 现象 | 原因 / 解决 |
|---|---|
| 完全收不到推送 | ① Xcode 没开 Push 权限(第 2 步)② device_tokens 没你的 token(第 4 步没进 Genius Bar / 没授权)|
| `device_tokens` 一直空 | App 没拿到 token:多半是 Push 权限/描述文件没生效;重装 App 再试 |
| 函数报错 / 推送失败 | Supabase → Edge Functions → `notify-user` → **Logs** 看报错;常见是 `APNS_KEY_P8` 没贴全(缺 BEGIN/END 行)或 Key ID/Team ID 填错 |
| **真机装的开发版收不到,但 TestFlight 能** | `APNS_HOST` 用错环境:**开发/调试版 = sandbox**,TestFlight/App Store = 正式。改 `APNS_HOST` 后重 deploy |
| 模拟器收不到 | 正常,APNs 真实推送只到真机(模拟器只能拖 .apns 文件模拟显示) |
| App 开着时不弹 | iOS 默认前台不弹横幅;聊天页/首页有 Realtime 即时刷新。要"前台也弹横幅+响声"需开发侧加前台展示处理(见下) |

## 附:不想搞 APNs 的轻量替代
如果你**只要"App 开着时"提醒**(不需要 App 关闭后的离线推送),完全可以不配 Apple 那一套:
- 用现有 **Realtime**:收到新客服消息时,App 本地发一条通知(`UNUserNotificationCenter` 本地通知)。
- **零 Apple 配置、模拟器也能测**,但 App 被杀死/长时间后台时收不到。
- 需要走这条就告诉我,我直接加。
