### Cloudflare 控制台一次性设置（用户操作）

- **Root directory**：`site`
- **Build command**：`npm run build`（构建前自动 `fetch` 拉最新 Supabase 快照）

## 三、安全审查与加固（本次）
结论：**无匿名/远程可利用漏洞** —— 无不可信用户输入到达 HTML 注入点；anon key 只读+RLS+不进客户端包；dist 无密钥/音频/完整 guides 数据泄漏；外链均带 `rel=noopener`。以下为纵深防御加固：

- **存储型 XSS（CMS 图文）**：`set:html` 渲染的 city.description / attraction.introduction|summary / sub_area.body / content_blocks.body 改为经 `clean()`（`sanitize-html`，**仅构建期运行，不进客户端包**）清洗——标签/属性白名单，剥离 `<script>`/`onerror`/`javascript:`，外链强制 `rel=noopener nofollow ugc`。
- **客户端 innerHTML（签证 widget）**：所有来自 Supabase 的标签字段（policy/city/port/country 名、flag、headline）经 `esc()` HTML 转义后再拼入 innerHTML。
- **安全响应头**：新增 `site/public/_headers`（CF 静态资源识别）：CSP（script-src 'self'，无内联脚本已自检通过）、X-Frame-Options DENY、X-Content-Type-Options nosniff、Referrer-Policy、Permissions-Policy、COOP。
- **依赖**：`npm audit` 仅 esbuild 一项（dev-server 限 Windows 的任意文件读，GHSA-g7r4-m6w7-qqqr）——**不影响生产静态站**，需 astro 7 破坏性升级才能消除，暂不动。
- 验证：清洗器拦截 `<script>`/`onerror`/`javascript:`，正常 `<p><strong>` 保留；115 页构建通过；dist 0 内联脚本/事件处理器（CSP 兼容）。

## 五、放票倒计时 + CSP 内联脚本修复（本次）
- **放票倒计时**：`ReleaseCountdown.astro` + `src/scripts/release-countdown.ts`，按景点 `release_time`（北京时间 HH:MM，固定 UTC+8 无夏令时，全球时区正确）倒计时到下一次放票；`advance_days` 推算"本批开放约 X 日后日期"。数据驱动：填了 `release_time` 才渲染。故宫填常见公开规则（20:00/提前 7 天）+「以官方为准」注脚，待核实微调。
- **校验器外置**：`PassportNameChecker` 脚本抽到 `src/scripts/passport-name-checker.ts`。
- **CSP 内联脚本修复（重要）**：发现 Astro 会把小体积脚本**内联**为 `<script type=module>`，而上一次安全提交(9e414d0)写在 `_headers` 的 `script-src 'self'` 会**屏蔽所有内联脚本**（含 hotels 复制按钮、新倒计时/校验器）。改为启用 **Astro `experimental.csp`**：每页 `<meta>` 注入 CSP 并**自动哈希**其内联脚本/样式 → `script-src 'self' 'sha256-…'`（无 unsafe-inline）。`_headers` 移除 CSP、保留 XFO 等。
- 配套：把 3 处内联 `style=` 改为工具类（内联样式属性无法哈希）；签证"具体行程"按钮初始隐藏改用 `.is-hidden` 类（运行时 `.style` 由 JS 改，CSP 不拦）。
- 验证：逐页校验每个内联脚本 sha256 均在该页 CSP 内（missing=0）→ 严格 CSP 下脚本可执行；0 残留内联 style 属性；119 页通过。

## 七、CMS 改动自动重建官网（webhook → CF deploy hook）
SSG 站点改 CMS 不重建不生效。做成「后台一改 → 自动重建」。

- 迁移 `091_site_rebuild_webhook.sql`：`pg_net` + `supabase_vault`；`public.notify_site_rebuild()`（SECURITY DEFINER）从 Vault 读 `cf_deploy_hook_url` 并 `net.http_post` 打 CF deploy hook；在站点相关表（ticket_attractions / cities / attractions / sub_areas / city_guides / culture_tips / visa_* 等）建 statement-level AFTER 触发器。
- **安全**：deploy hook URL 是密钥、且本仓库公开 → **不入库到仓库**，存 Supabase Vault（`cf_deploy_hook_url`）。Vault 未设则触发器空操作。
- 已实测 CF deploy hook 有效（POST 返回 success、build queued）。

### 启用（你操作）
1. SQL Editor 跑（**勿提交**，替换真实 URL）：`select vault.create_secret('https://api.cloudflare.com/.../deploy_hooks/XXXX', 'cf_deploy_hook_url');`
