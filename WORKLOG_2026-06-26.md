## 二、Cloudflare 自动部署（本次）
方式：**Cloudflare 控制台 Git 集成（Workers Builds）**，与 `yoloadmin-vue` 同款。不用 GitHub Action、不用 `CLOUDFLARE_API_TOKEN`。push 到 main 即由 Cloudflare 自动构建部署。

- 配置自包含在 `site/wrangler.jsonc`：assets-only Worker「`yolo-site`」，`directory: dist`，`html_handling: auto-trailing-slash`，`not_found_handling: 404-page`。
- （先前试过 GitHub Actions 方案，CI 构建成功但缺 `CLOUDFLARE_API_TOKEN` secret，按用户选择改用控制台 Git 集成，已删除 `deploy-site.yml` 与根 `wrangler.site.jsonc`。）
- `astro.config.mjs`：`site` 改读 `SITE_URL` 环境变量（CF 构建变量），自定义域名免改代码；`Base.astro` canonical/OG 跟随 `Astro.site`。
- 新增 `site/src/pages/404.astro` 配合 `not_found_handling`；`site/package.json` 加 `deploy` 脚本（本地 `npm run deploy`）。

### Cloudflare 控制台一次性设置（用户操作）
Workers & Pages → Create → Workers → 连接本 GitHub 仓库，设：
- **Root directory**：`site`
- **Build command**：`npm run build`（构建前自动 `fetch` 拉最新 Supabase 快照）
- **Deploy command**：`npx wrangler deploy`
- 之后 push 到 main 即自动构建部署到 Worker `yolo-site`。

### 内容更新如何上线
- 本质仍是 **SSG 构建期快照**（SEO/速度/成本最优）：HTML 是 build 那一刻的数据。
- 代码 push → CF 自动重建部署；构建会重新抓取 Supabase，顺带带上最新 CMS 内容。
- **纯 CMS 内容改动**（无 push）：在 Cloudflare 项目里点 **Retry deployment / Create deployment** 手动触发一次（约 1–2 分钟生效）；后续如需「改 CMS 自动重建」可加 Supabase webhook → CF deploy hook（未做，待定）。
