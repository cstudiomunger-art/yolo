## 二、Cloudflare 自动部署（本次）
方式：**GitHub Actions + wrangler**，镜像现有 `deploy-admin.yml`/`deploy-support.yml`（仓库已配 `CLOUDFLARE_API_TOKEN`，无需 dashboard 手动连接）。push 到 main 即自动部署。

- `wrangler.site.jsonc`：assets-only Worker「`yolo-site`」，`directory: site/dist`，`html_handling: auto-trailing-slash`，`not_found_handling: 404-page`（镜像 `wrangler.admin-vue.jsonc`）。
- `.github/workflows/deploy-site.yml`：push 触发（paths 限 `site/**` 等）+ `workflow_dispatch` 手动触发；Node 20 → `npm install && npm run build`（构建前自动 `fetch` 拉最新 Supabase 快照）→ `cloudflare/wrangler-action@v3 deploy`。
- `astro.config.mjs`：`site` 改读 `SITE_URL` 环境变量（repo variable），自定义域名免改代码；`Base.astro` canonical/OG 跟随 `Astro.site`。
- 新增 `site/src/pages/404.astro` 配合 `not_found_handling`。

### 内容更新如何上线
- 本质仍是 **SSG 构建期快照**（SEO/速度/成本最优）：HTML 是 build 那一刻的数据。
- 代码 push → 自动重建部署；构建会重新抓取 Supabase，顺带带上最新 CMS 内容。
- **纯 CMS 内容改动**（无 push）：在 GitHub Actions 页面点 `Deploy Site → Run workflow` 手动触发一次即可（约 1–2 分钟生效）。后续如需「改 CMS 自动重建」，可加 Supabase webhook → Cloudflare deploy hook（未做，待定）。
