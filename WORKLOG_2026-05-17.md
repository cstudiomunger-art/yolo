## 二、遇到的问题

### 1. 需求理解偏差

- 首次按「代码键 / CodingKeys」理解任务，与用户意图（**通读全部代码并对照文档**）不符。
- **处理：** 二次通读 37 个 Swift 文件 + 迁移 + admin，按模块输出「已完成 / 未完成」清单。

### 2. 工程命名与文档不一致

- Xcode 工程名为 **YOLO**，开发文档写的是 **ChinaGo**、`com.chinago.travel`。
- **影响：** 对照文档时容易误判「功能缺失」，实际是同一套代码、命名未统一。
- **建议：** 统一 Bundle ID / 显示名，或在 README 中写明对应关系。

### 3. 「远程内容」与「内置 JSON」并存

- `app_settings.use_remote_content` 开启后走 Supabase，但 `Resources/Static/*.json` 仍完整保留作 `FORCE_BUNDLED` / 离线回退。
- **与文档原则：** 上线内容不应写死在代码——当前架构支持远程，但需确保生产环境 CMS 数据完整，否则仍回落到 Bundle。
- **风险：** 运营误以为已切远程，实际 App 仍可能用缓存或 bundled 数据。

### 4. CMS 封面 URL 与 App 展示未闭环

- 后台上传封面后写入 `cover_image_path`（多为 Storage 公网 URL 或路径）。
- **App 端：** `GuideView` / `CityOverviewView` 等仍以占位色块为主，**未实现 `AsyncImage` + CDNRouter**。
- **结果：** CMS 可上传封面，用户在 App 里暂时看不到图。

### 5. 迁移执行顺序敏感

- `008_seed_content` 在 `010_fix_text_primary_keys` 之前执行会报 uuid 类型错误（README 已强调，仍易踩坑）。
- 新增 `013` 需在 Supabase SQL Editor **按 README 顺序**执行，否则封面上传报 bucket 不存在。

### 6. Admin 重构尚未提交、未做回归测试

- `git status` 显示 admin 大量改动为 **未提交** 状态；`admin.js` 从 ~780 行拆到多文件。
- **风险：** 城市工作台、封面上传、关联下拉等需在本地 `python3 -m http.server` 后逐表点击验证。
- 未确认：登录、列表、编辑、删除、音频上传、封面上传是否全部正常。

### 7. 核心能力仍为占位（审计结论，非今日新 bug）

| 能力 | 现象 |
|------|------|
| AI | `useRemoteAI=true` 时提示未连接；否则用 `assistant_replies` / 样本行程 |
| IAP | `useRemoteIAP=true` 时购买仅弹登录；无 RevenueCat |
| 音频 | 无 URL 时播放器走模拟进度；无后台播放 |
| 酒店/机票外链 | Link 多为固定 `booking.com` / Google Flights，未按文档实现带参 `buildHotelSearchURL` |
