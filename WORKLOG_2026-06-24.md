# WORKLOG 2026-06-24 · Plan 创建流程接入签证判定 + 友好路线推荐

把签证判定与"友好路线"推荐**前移进 AI 推荐行程的创建主流程**:选完城市/日期后、生成行程前,按用户护照当场判定本次行程签证状况;不够用时给出经引擎复核的推荐路线(加过境 / 换城 / 删城)让用户选择确认。并新增"替换一座城"策略。

## 背景
原流程 `选城市 → 选日期 → 生成 → 预览 → 保存` 全程不判签证;签证能力只在行程**存好之后**以详情页一条 `VisaPlanHintBanner` 横幅出现(不够用才飘),点开才进检测器。判定引擎 / 粗判 / 路线生成 / 对比卡其实都已存在,本次主要是**接入时机前移** + 补"换城"算法。

## 改了什么

### 1. 新增"换城"策略 — `Services/VisaTripChecker.swift`
- 新增私有 `swapBlockers(query:appCities:catalog:data:rec:)`:取 `rec.blockers`(拖累城)→ 保留非拖累城,用候选城池 `catalog`(未在行程、可映射 admin code、且单独成行引擎判绿的城,按人气降序)等量回填,整组再调 `recommend` **复核为绿**才产出。文案"把 X 换成 Y,城数不变,经引擎复核全程免签"。
- `routes(...)` 新增 `catalog: [(slug:String, popularity:Int)] = []` 入参;由"只产一条 friendly(`??` 短路)"改为**收集多条**:加过境 + 换城;两者皆无时回落删城(至多 2 条 friendly)。排序按损失:`加过境(不丢城) > 换城(等量) > 删城(丢城) > 办 L 签`。
- `catalog` 缺省空 → 行为退化为现状,检测器结论页调用无需改。transit/swap/drop 标题区分("加一城过境/换城/去掉拖累城")。

### 2. 粗判单一真源 — `Services/VisaCoarseCheck.swift`
抽出 `query(citySlugs:start:end:countryCode:data:) -> VisaQuery?`,`recommendation(...)` 改为调它。让"判定的 rec"与"生成路线用的 query"是**同一份**,避免重复构造漂移。

### 3. 路线卡组件可复用 — `Features/Visa/PlanRouteVisaCompareView.swift`
- 新增可选 `onAdopt: ((VisaRoute)->Void)?`:默认仍写 `preferences.selectedCityIds`(检测器路径不变),创建流程可传闭包改本地草稿。
- 采用状态由 `VisaRoute.Kind?` 改为 `UUID?`(`adopted == route.id`)——现在可能有多条 friendly,按 Kind 追踪会一采全亮。

### 4. 创建流程插入签证步(主改动)— `Features/Plan/PlanCreateFlowView.swift`
- `enum Step` 增 `.visa`;日期步"Generate itinerary"按钮改为先进 `.visa`(`enterVisaCheck()`),不再直接生成。
- `enterVisaCheck()`:
  - 护照国为空或 `CN` → 直接生成(**静默跳过**:本引擎判的是外国人入境中国,CN 护照游国内不该误报)。
  - `await visaData.load()`,用**真实** `arrivalDate/departureDate` 经 `VisaCoarseCheck.query` 建查询 → `recommend`。
  - `isEnough`(绿/够用)或无可判城市 → 直接生成(**静默放行**)。
  - 不够用 → `VisaTripChecker.routes(..., catalog: cities.map{($0.id,$0.attractionCount)})`,展示结论头 + 路线卡 + 两个出口(「按推荐确认并生成」/「保持原行程继续」)+「精细核对」入口(present 现有 `VisaDetectorView` preset)。
- `adoptVisaRoute`:采用换城/删城更新本地 `selectedCityIds` 后生成;采用加过境(港澳为信息性、不进内容行程)城市集不变。
- 新增 `visaStep` / `visaVerdictHeader` / `visaRouteCard` / `cityDisplayName`(沿用 Theme token,视觉对齐检测器对比页)。

## 验证
- **编译**:`xcodebuild -project YOLO.xcodeproj -scheme YOLO -destination 'generic/platform=iOS Simulator' build` → **BUILD SUCCEEDED**。
- **引擎/策略回归**:扩展 `scripts/visa_live_test.swift`(swiftc 真引擎 + 线上 8 政策/201 grant/115 城/920 矩阵/28 口岸):
  - 原 A/B/C 全量基准不变;
  - 新增 **C2**:IN 散客(每城都需签证、无绿候选)→ 换城优雅返回 nil,不出换城卡;
  - 新增 **D1**(合成集):含拖累城 → 引擎复核把"乙市"换成"丙市",城数不变全程绿;
  - 新增 **D2**:全可达 → `isEnough` → routes 为空(静默放行)。
  - 结果 **12 通过 / 0 失败**。
- 说明:线上当前 6 座内容城市同质(均在单方面免签 + 240h 区),换城在真实数据上暂不会触发;待内容城市扩到免签区外才会出现"换城"卡。新算法以合成集验证。
