# WORKLOG · 2026-06-25 · 官网（获客漏斗）v1

新建 `site/` —— 面向外国游客的 YOLO HAPPY 营销官网，定位为**获客漏斗**（网页非产品，App 才是；网页用高意图长尾 SEO + 当场可用微工具建立信任 → 全站常驻 “Get the app” 收口）。

## 决策（与用户确认）
- **技术栈**：Astro（静态生成，SEO 友好；交互工具用客户端脚本岛，零框架依赖）。
- **品牌**：网页锁新体系——暖白 `#FAF9F7` + 珊瑚橙 `#C45C26` + Poppins/Lora（App 现有沙金 `#C8A97E` 暂不动，后续再靠齐）。
- **签证工具**：自建 TS 引擎，构建期从 Supabase 拉数 → 快照打包；两层全做 + 按国籍静态 SEO 页。
- **范围**：v1 只做签证工具；故宫票 / 外宾酒店两页先做 SEO 占位。

## 数据管道
- `site/scripts/fetch-visa-data.mjs`：构建期用 publishable anon key 拉 7 张 `visa_*` 表（RLS：`is_active` 行 anon 可读）→ `src/data/visa-data.json`；复用 App 的 `ISO3166Countries.swift` 生成 `src/data/countries.json`（英文名 via `Intl.DisplayNames`，旗帜由 alpha-2 推导，slug 去音标规范化）。
- 跑通：policies=8 / grants=201 / cities=115 / matrix=920 / ports=68｜countries=248（withGrant=90）。
- 快照为 build 产物，已 gitignore，每次 `npm run build` 前自动刷新。

## 引擎移植
- `src/lib/visa-engine.ts`：忠实移植 `VisaPolicyEngine.swift`（GATE0 护照有效期 → S1 全候选资格 → S3 四维健康表 → scorePolicy 选最松通过 / amber 最小 gap + Plan A 换城市·B 办签 / red visa_L 兜底）。
- `src/lib/visa-summary.ts`：移植 `VisaNationalitySummary`，扩展输出适用政策明细（停留天数/条件/来源/有效期）供按国籍页用。
- 正确性抽检（生成页）：日本🟢免签30天 · 美国🟡有条件(240h过境) · 印度🔴需签 · 新加坡🟢互免，与 App 一致。

## 页面（Astro，105 个静态页）
- `/` 首页：楔子「Not an OTA. A system navigator.」+ 三痛点工具卡 + OTA 对比 + 信任三栏。
- `/china-visa-free-checker` ★ 英雄页：第一层选护照即时裁定；第二层「具体行程」展开（出发/续程/城市多选/日期/口岸/续程票/随团）跑完整引擎；底部按护照国家网格。
- `/china-visa-free/[country]`：90 个有 grant 国 + 16 个高流量需签市场，共 ~105 页，命中「can [国籍] travel to china visa free」。
- `/forbidden-city-tickets-foreigners`、`/china-hotels-that-accept-foreigners`：SEO 内容骨架 + 工具占位（酒店页含可复制双语确认模板）。
- `/about`：信任页，化解护照信任焦虑（官方来源 / 日期戳 / 同源引擎 / 不收数据 / 非 OTA 模式）。
- 全站 `GetTheApp` 常驻；`@astrojs/sitemap` + `robots.txt`。
