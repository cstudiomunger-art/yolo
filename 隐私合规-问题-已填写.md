## 二、隐私政策相关

1. **服务器部署地**：用户数据实际存储在哪个国家/地区的服务器？

答：

| 数据类型 | 存储/处理位置 | 服务商 |

| 行程分享页、邮箱确认页、重置密码页 | Cloudflare Workers 全球边缘节点（主域名 `yolo.cstudiomunger.workers.dev`） | Cloudflare |

2. **数据传输路径**：用户数据从客户端到服务器的完整链路经过哪些国家/地区？

答：

1. **常规业务数据**：用户 iOS 设备（全球）→ HTTPS → **Supabase（美国西部 us-west-1）**（注册/登录、行程同步、收藏、清单进度、头像上传等）
2. **AI 功能**：用户 iOS 设备 → Supabase Edge Function（美国西部）→ **火山引擎方舟大模型 API（中国北京）** → 返回 AI 生成结果；请求内容为**文本**（行程偏好、助手对话消息等），不上传照片给 AI 分析
3. **身份认证**：
   - Apple 登录：设备 ↔ Apple 服务器 ↔ Supabase Auth
   - Google 登录：设备 ↔ Google OAuth 服务器 ↔ Supabase Auth
   - 邮箱注册/登录：设备 ↔ Supabase Auth
4. **行程分享**：开启分享后，只读链接经 Cloudflare Workers 边缘节点访问 Supabase 公开读取接口
5. **推送通知**：Supabase/后端 → **Apple APNs** → 用户设备
6. **内购（IAP）**：设备 ↔ **Apple App Store**；订阅状态校验经 **RevenueCat**（美国）→ 回写 Supabase（接入完成后）

3. **数据保存期限**：用户账号注销后，你们会在多久内删除数据？有没有法定要求必须保留的？

答：

- **即时删除（业务层）**：用户在 App「设置 → 删除账号」确认后，调用 Supabase Edge Function `delete-account`，**立即**删除 `auth.users` 记录；数据库中关联表（`profiles`、`user_itineraries`、收藏、清单进度、客服会话、IAP 交易日志等）均设置了 `ON DELETE CASCADE`，随用户 ID 一并删除；用户头像文件（`avatars/{userId}/`）同步清理。
- **平台备份保留**：Supabase 平台级数据库备份可能额外保留 **7～30 天**（取决于 Supabase 订阅套餐），属基础设施备份，非业务可访问数据。
- **法定保留**：
  - **Apple IAP 交易**：Apple 依法保留交易收据；若接入 RevenueCat，RevenueCat 侧亦有其数据保留政策。
  - **财务/税务**：如产生境内应税收入，会计凭证、发票等按《中华人民共和国会计法》《税收征收管理法》等要求保留（通常不少于 10 年），可与个人账号数据分离、匿名化处理。
  - **退款申请记录**：当前与用户账号 CASCADE 删除；若未来需审计，可改为注销后匿名化保留。

## 五、AI 内容合规声明

1. **你的小程序用到 AI 的具体方式是**：（本产品为 iOS App）

答：

| 维度 | 说明 |
|-----|------|
| 是否上传照片给 AI 分析 | **否**。AI 仅处理**文本输入**（行程偏好、天数、城市、助手对话消息等），不上传用户照片进行 AI 视觉分析 |
| 本地 vs 云端 | **云端为主 + 本地兜底**：登录用户调用 Supabase Edge Function → 火山引擎方舟大模型；未登录或远程失败时，行程规划走**本地确定性算法**（`PlanItineraryAssembler`） |
| 模型部署位置 | **火山引擎云端**（中国北京节点），非端侧本地模型 |
| 主要 AI 场景 | ① AI 辅助行程规划 ② 旅行助手文本对话 ③ 客服消息自动翻译 |

2. **AI 生成的内容**（分析结果/建议）是否有可能在用户间分享/传播？

答：

| 内容类型 | 可见范围 |
|---------|---------|
| AI 生成的行程 | **默认仅本人可见**；用户主动开启「分享链接」或「分享图片」后，行程内容可被持有链接/图片的任何人只读查看；用户可随时关闭分享 |
| AI 助手对话 | **仅本人可见**，不公开、不跨用户传播 |
| 客服聊天（含 AI 翻译） | **仅该用户与客服团队**可见 |

AI 生成内容为旅行信息参考，**非专业法律/医疗/签证意见**，已在服务条款中声明。

3. **用户是否可以拒绝 AI 处理**？（即是否提供非 AI 的替代功能路径）

答：**可以**。

- 游客（未登录）不调用远程 AI，行程规划使用本地算法。
- 登录用户远程 AI 失败时自动降级为本地行程组装。
- 助手场景可配置为 `offline` 模式，返回离线帮助指引。
- 音频导览、签证查询、行前清单、紧急信息、支付指南等**核心功能均不依赖 AI**，可完全非 AI 使用。

4. **内容审核机制**：是否有自动化过滤（如色情/暴力识别）？

答：

| 内容类型 | 审核方式 |
|---------|---------|
| 用户头像 | **人工审核**（后台 CMS 标记 `avatar_status`：pending / approved / rejected） |
| AI 生成文本 | **暂无**自动化色情/暴力/违法内容过滤；依赖大模型自带安全策略 + 业务 Prompt 约束 |
| 静态内容（景点、攻略等） | **人工编辑**（CMS 后台发布，非 UGC 自动生成） |
| 客服聊天图片 | **人工审核**（客服工作台查看，无自动图像审核 SDK） |

1. **争议解决方式**偏好：仲裁还是诉讼？如果诉讼，管辖法院选哪里？

答：**诉讼**（推荐）。

2. **账号注销政策**：用户如何注销账号？注销后多久生效？

答：

- **注销路径**：App → 个人中心 → 设置 → **Delete account（删除账号）** → 二次确认
- **生效时间**：确认后**立即生效**——调用 `delete-account` Edge Function，实时删除 Supabase Auth 用户及关联云端数据
- **不可恢复**：注销后账号、云端行程、偏好设置等**永久删除**，无法恢复
- **本地数据**：App 本地缓存需用户通过「清除缓存」手动清理，或卸载 App
