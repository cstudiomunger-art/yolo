## 一、今日工作内容

### 1. 行前清单模块（`checklist_items`）全字段 CRUD

对照 Supabase `020_checklist_v2.sql` 与 App 模型 `ChecklistItem`，完善 CMS 对 v2 字段的增删改查能力。

| 改动文件 | 说明 |
|----------|------|
| `admin/js/schema.js` | 扩展 `checklist_items` 表单与列表列；新增 `enum_multi` 展示标签；`showWhen` 按类型显示城市字段；类型/阶段/优先级中文标签 |
| `admin/js/core.js` | `checklistItemMatchesCity`、`filterChecklistRowsByCity/ByType`、`sanitizeChecklistPayload`、`getChecklistCreateDefaults` |
| `admin/js/crud.js` | 类型筛选工具栏；保存时清单 payload 规范化；`mountChecklistTypeVisibility`；宽弹窗 |
| `admin/js/field-types.js` | `enum_multi` 渲染与读取；列表单元格 `checklist_type/phase/priority` 格式化 |
| `admin/js/city-hub.js` | 城市工作台「行前清单」Tab 对齐主列表（编辑+删除、类型筛选、v2 城市匹配） |
| `admin/js/admin.js` | `checklist_settings` 走单页表单；切换表时恢复类型筛选 session |

**字段覆盖（均可编辑）：**

- 类型 `type`（entry / universal / city）
- 阶段 `phase`、优先级 `priority`
- 分组 `group_title`、标题 `title_en`
- 目标国籍 `target_nationalities`、目标城市 `target_cities`
- 为何重要 `why_important`、如何完成 `how_to_complete`（富文本）
- 外链 `external_links`（`link_list` 增删行）
- 预计分钟 `estimated_minutes`
- 展示标签 `display_tags`（Key / Urgent 多选）
- 文化提示 `cultural_tip`（富文本）
- 排序 `sort_order`、启用 `is_active`
- ID（高级选项 / slug）

**保存逻辑：**

- `entry` / `universal`：自动清空 `city_id`、`target_cities`
- `city`：同步 `target_cities` 与 legacy `city_id`；城市工作台新建时默认绑定当前城市

### 3. 富文本无法在弹窗中编辑（Bug 修复）

**现象：**「为何重要」「如何完成」「文化提示」等 `richtext` 字段显示内容但无法输入。

**根因：** `openModal` 在 `document.body.appendChild(backdrop)` **之前** 调用 `mountFieldInteractions` → `initRichTextHosts`。表单未挂载 DOM 时 `isHostVisibleForQuill` 为 false，Quill 标记 `quillPending` 后从未初始化（`bindDeferredQuillHosts` 仅监听子区域 `details` 折叠）。

**修复：**

| 改动文件 | 说明 |
|----------|------|
| `admin/js/crud.js` | 先 `appendChild` 再挂载交互；`ensurePendingQuillHosts`；关闭/保存时 `destroyQuillInRoot` |
| `admin/js/rich-text.js` | `bindDeferredQuillHosts` 增加 `requestAnimationFrame` 补挂待初始化编辑器 |
| `admin/index.html` | 静态资源版本号升至 `v=21`（避免浏览器缓存旧脚本） |

## 二、遇到的问题

### 1. v1 / v2 城市字段并存

- 库表同时有 `city_id`（legacy）与 `target_cities`（v2）。
- **处理：** 保存时 `sanitizeChecklistPayload` 按 `type` 统一；列表与城市筛选用 `checklistItemMatchesCity`（含 `target_cities`）。

### 2. 富文本「看起来像只读」

- 非 Quill 故障类配置问题，而是初始化时序问题。
- **验证：** 修复后弹窗内应出现 Quill 工具栏（加粗、列表、链接等），且可聚焦输入。

## 四、建议自测步骤

1. 硬刷新 CMS（`?v=21`），登录管理员账号。
2. 侧栏 **行前清单** → 新建/编辑一条，确认富文本可编辑并保存。
3. 切换 **类型** 筛选，确认 entry / universal / city 列表正确。
4. 侧栏 **清单设置** → 修改提醒天数与横幅模板 → 保存。
5. **城市工作台** → 某城市 → **行前清单** Tab → 新建/编辑/删除。
6. App Prepare 页拉远程清单，确认新字段（若已配置远程内容）展示正常。
