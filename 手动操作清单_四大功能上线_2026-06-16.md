## 阶段 A · 后端上线(让数据库和函数生效)

### A1. 把迁移推到 Supabase ⭐ 最先做
新增的 4 个迁移(062–065)目前只在代码里,数据库还没有这些表。

```bash
cd /Users/vesperal/Desktop/YOLO
# 首次需登录 + 关联项目(已登录可跳过 login)
supabase login
supabase link --project-ref edwvrriuwzaaqznklrgi
# 预览将要执行的迁移
supabase db push --dry-run
# 确认无误后真正执行
supabase db push
```
- **为什么**:不推迁移,App 里签证/支付/客服/交通方言全部读不到数据(会回退到 App 内置的兜底种子,功能能用但后台改不了)。
- **验证**:Supabase Dashboard → Table Editor,应看到 `visa_policies / visa_policy_grants / city_visa_tags / entry_ports / visa_rule_overrides / payment_advice_rules / payment_merchant_phrases / payment_helper_links / support_agents / support_conversations / support_messages / transport_tips / common_phrases / dialect_phrases` 这些新表,且有种子数据。
- **若没有 Docker / 不想用 CLI**:可在 Dashboard → SQL Editor 里,按顺序把 `supabase/migrations/062…065.sql` 四个文件内容**逐个粘贴执行**。

### A2. 部署翻译 Edge Function `chat-translate`
```bash
supabase functions deploy chat-translate
```
- **为什么**:客服聊天的自动翻译靠它。没部署的话,消息照常收发,但不翻译。
- **验证**:
```bash
# 用你的 anon key 测一下(替换 <ANON_KEY>)
curl -s -X POST "https://edwvrriuwzaaqznklrgi.supabase.co/functions/v1/chat-translate" \
  -H "Authorization: Bearer <ANON_KEY>" -H "apikey: <ANON_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"text":"I cannot bind my card","target_lang":"zh"}'
# 期望返回 {"translated":"我无法绑定我的卡", ...}
```
- **关于翻译用哪家**:函数默认**复用你 ai-complete 已配的火山引擎 key**(`VOLCENGINE_API_KEY`,Supabase 函数密钥是项目级共享的),所以**通常无需额外配置**就能用。

### A3.(可选)在后台改用别的翻译模型/Key
若想让翻译走另一个模型或独立 key,而不是默认火山引擎:
- 打开 admin CMS → 全局配置 → 应用配置(app_settings),填写以下任一字段(留空=用默认):
  `translate_model` / `translate_api_url` / `translate_api_key` / `translate_target_default`
- **为什么**:满足「翻译 provider 后台可配」。不填就用默认,完全没问题。

## 阶段 B · 数据核对与配置(上线前必做)

### B1. 签证 6 国数据 —— 法务/政策核对 ⚠️ 重要
迁移里 `visa_policy_grants` 的 6 国(英美法德新日)政策与生效窗口是**起步示意数据,需核对**(尤其 P2 单方面免签的 `expiry_date` 会随政策延期变动)。
- 打开 admin CMS → 🛂 签证引擎 → 政策适用(国籍×窗口)、政策框架、入境口岸、城市签证标签。
- 逐条对照官方最新签证政策修正;**新增更多国家**也在这里加(其余国籍 App 会引导「问真人确认」)。
- **为什么**:签证判定直接影响用户是否买机票出行,错判风险高。这是合规红线。

### B2. 填客服「能帮 / 不能帮」边界文案 —— 法务确认
- App 端 Genius Bar 主页底部那段「我们能帮 / 暂不能帮」目前是占位文案(代码里标了「待团队最终确认」)。
- 定稿后告诉我,我改进代码(或后续做成 CMS 可配)。
- **为什么**:涉及医疗/法律免责,需法务把关。

### B3.(可选)上传方言/常用语真人音频
- admin CMS → 🧭 实用信息内容 → 常用语 / 方言彩蛋,在 `音频 URL` 字段填入音频公开链接。
- 音频先上传到 Supabase Storage 的 `audio-guides` bucket(公开),拿到 URL 后填入。
- **为什么**:不填则 App 用系统 TTS 念普通话兜底(能用,但不是真人方言味)。

## 阶段 C · 建客服坐席账号

### C1. 为每个团队客服建登录账号
1. 让每位客服用邮箱在 App 或 web/support 登录页**注册一次**(走 Supabase Auth),或你在 Supabase Dashboard → Authentication → Users → Add user 手动建。
2. 在 Dashboard → Authentication → Users 里复制每个客服的 **User UID**。

### C2. 在 CMS 把账号登记为坐席
- admin CMS → 💬 客服管理 → 客服坐席 → 新建:
  - `Auth 用户 UUID`:粘贴 C1 的 User UID(**必填,决定谁能登录工作台**)
  - `姓名` / `线路/专长`(如「北京 · 签证」)/ `头像种子`(首字,如「程」)/ `语言`(如 中文、英文)
  - `状态`留 offline(客服上线时自己在工作台切换)
- **为什么**:只有 `support_agents` 里有记录的 auth 用户,才能登进客服工作台、才会出现在 App 用户的客服 grid 里。
- **验证**:App 端 实用信息 → Genius Bar,应看到这些客服头像。

## 阶段 D · 部署 web/support 客服工作台

### D1. 配置 Cloudflare 部署密钥(一次性)
- GitHub → 仓库 Settings → Secrets and variables → Actions → New repository secret:
  - `CLOUDFLARE_API_TOKEN`(Cloudflare → My Profile → API Tokens → Create Token,模板「Edit Cloudflare Workers」)
  - (可选)`SUPABASE_URL`、`SUPABASE_ANON_KEY`(不填则用代码里的公开默认值)
- 配好后:**已推送的代码会自动触发 `Deploy Support Console` workflow**(也可在 Actions 页手动 Run)。
- **手动部署替代**(本机):

- **验证**:Cloudflare → Workers & Pages → `yolo-support` 拿到 URL,打开 → 用 C1 的客服账号登录 → 顶栏点「上线」→ App 端该客服变绿点。

## 阶段 G · 客服离线推送 APNs(可选,要让"App 关着也收到客服回复")

> 不配也不影响:App 开着时一切正常,只是关掉后收不到推送。配了才有离线推送。

### G1. 迁移已含 `device_tokens` 表
`066_support_chat_commercial.sql` 已在阶段 A1 一并推送(若 A1 在本次之前做的,需再 `supabase db push` 一次补 066)。

### G2. Xcode 开 Push 权限
- Xcode → Target YOLO → Signing & Capabilities → **+ Capability → Push Notifications**。
- 确认 `YOLO.entitlements` 出现 `aps-environment`。
- **为什么**:没有这个权限,App 拿不到 device token,推送链路是断的(代码已写好,会优雅降级不崩)。

### G3. 创建 APNs Key 并设为函数密钥
- Apple Developer → Certificates, Identifiers & Profiles → Keys → **+** → 勾选 **Apple Push Notifications service (APNs)** → 下载 `AuthKey_XXXX.p8`(只能下一次),记下 **Key ID** 和 **Team ID**。
- Supabase Dashboard → Project Settings → Edge Functions → Secrets,添加:
  - `APNS_KEY_P8` = .p8 文件全文(含 BEGIN/END 行)
  - `APNS_KEY_ID` = 10 位 Key ID
  - `APNS_TEAM_ID` = 10 位 Team ID
  - `APNS_BUNDLE_ID` = `com.chengduyuliu.yolohappy`
  - `APNS_HOST` = `api.sandbox.push.apple.com`(开发/TestFlight)或 `api.push.apple.com`(App Store 生产)

### G4. 部署推送函数
```bash
supabase functions deploy notify-user
```
- **验证**:真机(模拟器收不到推送)登录 App → 打开一次 Genius Bar(触发注册 token)→ 用 web 工作台给该用户回复 → 锁屏应收到推送横幅。
