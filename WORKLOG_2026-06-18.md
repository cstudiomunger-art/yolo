## 景点详情页：名称/地址一键复制

### 需求
景点详情页的名称、地址后面加复制按钮，点击即可复制。

### 实现
- 文件：`YOLO/Features/Guide/AttractionDetailView.swift`（`titleBlock`）
- 在**中文名**（`chineseName`）那行右侧加小复制图标 → 复制中文名
- 在**中文地址**（`addressZh`）那行右侧加小复制图标 → 复制中文地址
- 复制走 `UIPasteboard.general`，点击后图标由 `doc.on.doc` 变为绿色 `checkmark`，1.5 秒后自动还原
- 新增 `CopyField` 枚举 + `copiedField` 状态区分两个按钮的反馈，`copyButton(text:field:)` 复用按钮视图
- 中文名为空时不渲染该行（顺手补的判空保护）

### 设计选择（已与用户确认）
- 名称按钮复制**中文名**、地址按钮复制**中文地址**（方便给司机/当地人看、打车导航）
- 样式：小图标按钮（非图标+文字）

### 验证
- `xcodebuild -scheme YOLO -destination 'generic/platform=iOS' build` → **BUILD SUCCEEDED**

## 选择城市页：城市卡片显示真实封面图

### 问题
"Choose cities" 页城市卡片只显示 📍 图钉，没有图片。

### 诊断
- 卡片代码原本只渲染 `Text(city.emoji ?? "📍")`，**从未渲染任何照片**，📍 是 emoji 字段为空时的兜底符号。
- 线上 `cities` 表 `emoji` 被清空 → 全部退回 📍；`cover_image_path` 种子数据本就是 NULL 且代码未使用。
- admin 后台 `schema.js` 早已支持给城市上传封面（`cover_image_path` + 上传到 `cities` 文件夹），仅 iOS 端缺渲染。

### 实现
- 文件：`YOLO/Features/Plan/PlanCreateFlowView.swift`（`cityGrid` / 新增 `cityCover`）
- 卡片重构为「顶部封面 + 下方名称」：
  - 有 `coverImagePath` → 用 `CoverImageView` 显示真实城市照片（高 88）
  - 无封面 → ZStack 退回 emoji 占位（保持卡片美观）
- 用户后续在 admin「城市」里逐个上传封面即可生效（数据侧无需改代码）

### 验证
- `xcodebuild -scheme YOLO -destination 'generic/platform=iOS' build` → **BUILD SUCCEEDED**
