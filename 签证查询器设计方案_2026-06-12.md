# 签证查询器（Visa Checker）设计与布局方案

> 依据：2026-06-12 早十点线上会议纪要 · 签证功能定稿
> 状态：方案已评审通过，待开发

## 一、背景与约束

会议定稿：做「签证检查器」（独立入口，随时可查）+ 路线推荐双方案
（纯兴趣 / 签证友好），确认后进 Plan 行程列表再进行前准备。
规则研究已完成，算法与嵌入位置由开发侧定。

**现状关键约束**：现有 `visa_rules`（Supabase 表 + bundled JSON + `VisaRule` 模型）
**只有国籍维度**（免签与否 + 停留天数 + 文案 details[]）。会议要的「删掉某城市变免签」
本质是 **240 小时过境免签的城市适用区域逻辑**，必须给规则加城市维度。

---

## 二、数据与算法层

### 1. 规则模型扩展（单表扩展，不建新表）

`visa_rules` 加 4 列（migration `062_visa_transit_policy.sql`，admin `schema.js` 同步）：

| 新列 | 类型 | 说明 |
|------|------|------|
| `transit_hours` | INT | 240 = 10 天；NULL = 该国籍无过境免签资格 |
| `transit_city_ids` | TEXT[] | 适用城市集合（对应 cities.id），内容侧录入 CMS |
| `transit_note` | TEXT | 条件说明：须第三国进出、口岸限制等（展示用） |
| `visa_apply_url` | TEXT | 办签证官方/攻略链接（结果页行动区用） |

`VisaRule` Swift 模型（`TravelSafetyModels.swift`）同步加可选字段，旧数据解码兼容
（decodeIfPresent）；bundled `visa-rules.json` 补字段作离线兜底。

### 2. 判定引擎：纯函数 `VisaVerdictEngine`（新文件 `YOLO/Services/VisaVerdictEngine.swift`）

输入：`VisaRule` + 行程城市 ids + 行程天数 → 输出四态：

```
✅ visaFree(stayDays)            全国免签且天数够
⚠️ visaFreeOverstay(stayDays)    免签但行程超天数 → 提醒
🕐 transitEligible(hours, note)  城市全在适用区 & 天数达标 → 过境免签（附第三国进出条件）
🛂 visaRequired(blockers, 建议)   blockers 非空 = 删掉这些城市可转过境免签
```

纯函数、无 IO，配单测（规则正确性是这个功能的命门）。

**「签证友好路线」生成方式：本地规则调整，不二次调 AI**——
在同一次 AI 生成的路线上，由引擎找出 blocker 城市，移除后用现有
`PlanItineraryAssembler.build` 重排天数生成对照路线。
快、零额外 AI 成本、两条路线可比性强、结果可测。

## 三、UI 布局（三个触点）

### 触点 1：独立签证检查器（主交付物）

**入口两处，指向同一页面：**
1. 首页 Quick Access 两格改三格：`📋 Prep | 🛂 Visa Check | ⚠️ Emergency`，
   Visa 格副标题动态显示当前国籍判定（如 "🇬🇧 Visa-free"）
2. Hero 面板已有的签证摘要行加点击，进同一检查器

**页面形态**：全屏 modal（`AppModal` 加 `.visaChecker`，与 Prepare/Emergency 同模式），
新文件 `YOLO/Features/Visa/VisaCheckerView.swift`。
单页滚动、**输入齐即实时出结果，无「提交」按钮**：

```
┌──────────────────────────────┐
│ Visa Checker            Close│
│                              │
│ 护照国籍  [🇬🇧 United Kingdom ›]   ← 默认带入已选国籍，可换
│                                     （复用 passport_countries 选择器）
│ 检查行程                      │
│ [我的行程 ▾] / [自选城市]      ← segmented：已保存行程 / 手动选城市+天数
│                              │
│ ╭──────────────────────────╮ │
│ │ ✅ 免签 30 天             │ │  ← 结果大卡，三态配色：
│ │ 你的 10 天行程无需签证      │ │     ✅绿 / 🕐琥珀(过境免签有条件) / 🛂红(需办)
│ │ ─────────────────        │ │     大圆角，作 iOS 27 新风格首个试点
│ │ 依据：30-day visa-free…   │ │  ← rule.headline + details[] 列表
│ ╰──────────────────────────╯ │
│                              │
│ 行动区（按结果动态出现）：       │
│ · [查看免签调整建议]    （需办签证时，列 blocker 城市与删除后效果）
│ · [如何办理 L 签证 ↗]  （需办签证时，visa_apply_url）
│ · 第三国进出条件说明卡  （过境免签时）
│ · [加入行前准备清单]            │
└──────────────────────────────┘
```

### 触点 2：创建行程双路线（PlanCreateFlowView 加一步）

现有流程 `cities → dates → generating → review → save`，
在 generating 与 review 之间插**条件性步骤 `.routeChoice`**
（仅当判定为「需办签证且存在免签替代」时出现，否则直进 review）：

```
┌──────────────────────────────┐
│ 选择你的路线                   │
│ ╭──────────────────────────╮ │
│ │ ① 纯兴趣路线               │ │
│ │ 北京·成都·乌鲁木齐 · 10天    │ │
│ │ 🛂 需提前办理 L 签证        │ │
│ │              [选这条 →]    │ │
│ ╰──────────────────────────╯ │
│ ╭──────────────────────────╮ │
│ │ ② 签证友好路线 ⭐ 推荐      │ │
│ │ 北京·成都·西安 · 10天       │ │
│ │ ✅ 符合 240h 过境免签       │ │
│ │ 差异：移除乌鲁木齐，换入西安   │ │
│ │              [选这条 →]    │ │
│ ╰──────────────────────────╯ │
└──────────────────────────────┘
```

- ② 由本地引擎 + Assembler 生成；V1「换入城市」先只做移除，
  补位推荐（从免签区选热门城市）作可选增强，实现简单就带上
- review 页顶部固定加一行签证结论横幅（含状态色），所有行程都显示——
  免签也给 ✅ 正反馈（会议强调"极端有用性 / 一目了然"）

### 触点 3：已保存行程联动

- Plan 列表行程卡 + 详情页加签证状态徽章（✅/🕐/🛂），
  引擎实时判定，城市来源复用 `PlanTripCities.cityIds`
- 行前准备「办理签证」项走现有 checklist entry 类型按国籍 target
  机制（`target_nationalities`），**纯内容侧 CMS 配置，零代码**；
  检查器结果页「加入行前准备清单」按钮跳转 PrepareView 对应分组
