# WORKLOG · 2026-06-05

## 主题：付费墙（Paywall）不生效排查 + 解锁 UI 改版

### 一、问题
免费用户登录后，景点详情页的音频、正文全部解锁，付费墙不出现。

### 二、根因定位（判定链）
付费墙是否对某段内容生效，取决于两层门控：

1. **全局开关** `ContentModeService.effectiveUseRemoteIAP`
   = `isSupabaseConfigured && !forceBundled && !useMock && useRemoteIAP`
   其中 `useRemoteIAP` 来自远程 `app_settings.use_remote_iap`。
2. **单项开关** `PurchaseService.hasContentAccess(...)`
   首行 `if !requiresPurchase { return true }` —— 景点/子景点的 `requires_purchase` 为 false 即视为免费。

排查结果：
- 运行环境 `USE_MOCK=false`、`FORCE_BUNDLED=false`、Supabase 已配置 → 前三项 OK。
- **`app_settings.use_remote_iap = false`（真正根因）** → `effectiveUseRemoteIAP=false` → 所有内容全局放行，与会员/单项设置无关。
- 另：迁移 055 把 `attractions/sub_areas.requires_purchase` 默认设为 FALSE，且历史上从未有数据把它置 TRUE → 即便全局开关开着，也没有任何景点被纳入付费墙。
- 代码层 `use_remote_iap → useRemoteIap` 的 `convertFromSnakeCase` 解码链路确认无 bug。

### 三、数据修复（需在 Supabase SQL Editor 执行，线上数据操作）
1. 新增迁移 `supabase/migrations/058_paywall_all_content.sql`：
   把所有 `attractions` 与 `sub_areas` 的 `requires_purchase` 置 TRUE，缺价格档的补默认档 `single_t1`。（用户已执行）
2. 仍需手动执行（或后台「应用设置」打开）：
   `UPDATE app_settings SET use_remote_iap = TRUE WHERE id = 'global';`
   注意：后台保存 app_settings 时可能把该开关写回 false，需确认后台勾选「启用付费墙」。

### 五、后台管理系统能否真实修改景点付费效果——核查结论
能，但需同时满足 4 个前提：
1. **功能完整**：`admin/js/pricing-hub.js`「景点定价」页支持逐行/批量切换付费与价格档，直接写 `attractions/sub_areas` 的 `requires_purchase`、`price_tier_id`，列名表名正确。
2. **RLS 正确**：`attractions`、`sub_areas`、`app_settings` 均为「仅 `admin_users` 中的登录用户（`is_admin()`）可 UPDATE」（`005_admin_policies.sql`、`022`）。→ 前提：后台登录账号须在 `admin_users` 表内，否则写入被拒（界面弹错误 toast）。
3. **全局开关须先开**：`use_remote_iap=false` 时一切单项付费设置被忽略，必须先开全局。
4. **App 端 24h 磁盘缓存**（`ContentCacheStore`，stale-while-revalidate）：景点/子景点 JSON 一旦有缓存即先返回缓存；<24h 当次不刷新，>24h 先返回旧值并后台拉新（下次进入才生效）。即时验证需重装 App / 清缓存。
