## 一、登录/注册界面重设计

### 1. 新增共享样式组件 `YOLO/Features/Auth/AuthUI.swift`
- `AuthFieldStyle`：米白底（backgroundSubtle）+ 1pt 边框 + 4pt 圆角输入框，对齐 CountryPickerView 搜索框先例。
- `AuthPrimaryButtonStyle`：全宽 48 高、白底黑字 1pt 描边主按钮；禁用态降为 textDisabled；按压态米白底。
- `AuthDivider`：两侧细线 + 居中小字分隔（「或使用邮箱登录」）。
- `LegalLinksText`：一句话协议文案，内嵌 markdown 链接（`yolo://legal/terms|privacy`），`OpenURLAction` 拦截后以 sheet 弹出 `LegalDocumentView`——彻底解决旧版协议行「中间断开 + 两个 > 箭头」问题，服务条款/隐私政策两词金色可点。
- `legalDocumentSheet(item:)` 扩展：底部弹窗展示法律文档。

### 2. 重写 `LoginView.swift`（Form → ScrollView 自定义布局）
- 布局自上而下：Apple 按钮 → 协议小字 → 分隔线 → 邮箱/密码（注册多一个确认密码）→ 忘记密码（仅登录）/ 勾选协议行（仅注册）→ 主按钮 → 状态文案 → 底部模式切换。
- 去掉顶部 Sign In / Sign Up 分段选择器，改为页底「还没有账号？注册 / 已有账号？登录」（金色可点）。
- 注册按钮文案改为「Create Account / 创建账号」。
- 主按钮 loading 时内嵌菊花；邮件确认等待页改为米白卡片样式。
- **全部业务逻辑零改动**（提交校验、注册后自动登录、邮件确认状态机、埋点、错误转换）。

### 3. Apple 注册登录
- `AppleSignInButton`：`.signIn` → `.continue`，文案变「通过 Apple 继续 / Continue with Apple」，注册+登录一体（signInWithIdToken 首次即自动建号）。
- Apple 按钮下方新增一行协议小字「继续即表示你同意服务条款与隐私政策」（两处可点，sheet 查看），邮箱注册仍保留显式勾选。

### 4. 顺带统一 `ForgotPasswordView` / `SetNewPasswordView`
- 同样从 Form 改为自定义布局，复用 AuthUI 组件；60s 冷却、状态机逻辑不变。

### 5. `AuthLandingView`
- Logo 区顶部留白 40→48、底距 24→28，适配新表单。

### 6. 本地化 `Localizable.xcstrings`
- 新增 zh-Hans：两条协议 markdown 句、「还没有账号？」「已有账号？」「创建账号」「忘记密码？」「登录后开始规划你的中国之旅」。

## 二、Google 登录 + 布局再调整

### 1. Google 登录（OAuth 网页流程，零新依赖）
- 新建 `YOLO/Features/Auth/GoogleSignInButton.swift`：supabase-swift 内置 `signInWithOAuth(provider: .google, redirectTo: yolohappy://auth-callback)`，走 `ASWebAuthenticationSession`，无需引入 Google SDK、无需注册 URL Scheme。
- 用户取消（`canceledLogin`）静默处理不报错；成功埋点 `sign_in_google`。
- 按钮样式：白底 + 1pt 黑描边 + 蓝色 G + 「通过 Google 继续」，与 Apple 按钮同高同圆角（Apple 按钮 clip 成 8pt 圆角统一）。
- **后台前提**（未配则点击报错）：Google Cloud Console 建 OAuth Web 客户端（授权重定向 URI 填 `https://<项目ref>.supabase.co/auth/v1/callback`）→ Supabase Dashboard → Auth → Providers → Google 填 Client ID/Secret 启用 → Auth → URL Configuration → Redirect URLs 加 `yolohappy://auth-callback`。

### 2. 布局调整
- 登录模块（Apple/Google/邮箱整体）下移 72pt；Logo 区顶距 48→64。

## 三、登录页改为「三按钮选择页」+ 邮箱自动注册

### 1. 第一屏三按钮选择
- `LoginView` 重构为纯选择页：Apple / Google / 邮箱三个**统一白底浅灰描边**按钮（8pt 圆角，48 高）。
  - Apple 按钮由 `.whiteOutline`（深边）改 `.white` + 自绘浅灰描边，与 Google/邮箱完全统一（`SocialButtonChrome` modifier）。
  - 邮箱按钮：envelope 图标 + 「使用邮箱登录」，点击弹 `EmailAuthSheet`。
- 布局：Logo 顶部（顶距 80）+ `Spacer` 把三按钮推到屏幕底部。
- **显式协议同意**：三按钮下方放圆圈勾选行「我已阅读并同意 服务条款 与 隐私政策」（两词金色可点弹 sheet）；未勾选时三按钮 `.disabled` + 置灰（opacity 0.4），勾选后才可点。取代原「继续即表示同意」隐式小字。

### 2. 邮箱自动注册（新建 `EmailAuthSheet.swift`）
- 底部 sheet 弹窗，单一「继续」流程：先 `signIn`，失败且为 invalid-credentials 则 `signUp` 自动注册。
  - 已注册但密码错（signUp 返回混淆 user，`isAlreadyRegisteredUser`）→ 提示「邮箱或密码错误」。
  - 已注册未确认邮箱（signIn 返回 email-not-confirmed）→ 进确认提示。
  - 新注册成功且需确认 → 弹「账号已创建，请前往邮箱点击确认链接」（保持邮箱验证，防滥用）。
- 密码框 `AuthPasswordField`：单框 + 眼睛图标显示/隐藏切换（合并流程无法用「确认密码」防错）。
- 内含邮箱、密码、忘记密码（push）、继续按钮。

### 3. 文案
- 新增 zh-Hans：使用邮箱登录、返回、显示/隐藏密码、「输入邮箱和密码…自动创建账号」、「账号已创建…确认链接」、「请前往邮箱点击确认链接以完成登录」。
