# YOLO HAPPY 官网 · 项目总览

> 面向外国游客的获客漏斗官网。生成日期：2026-06-26。
> 详细分日记录见 `WORKLOG_2026-06-25_官网获客站.md`、`WORKLOG_2026-06-26.md`。

## 一、定位
- **网页不是产品，App 才是**。官网是**获客漏斗**：用高意图长尾 SEO + 当场可用的微工具建立信任 → 全站常驻「Get the app」收口。
- 差异化楔子：**「不是 OTA，是系统导航器」** —— 不卖票/签证/酒店，只翻译中国的规则，让用户自己操作本地系统。
- 受众英文优先。品牌锁：暖白 `#FAF9F7` + 珊瑚橙 `#C45C26` + Poppins/Lora。

## 二、技术栈与部署
- **Astro**（静态生成 SSG），目录 `site/`。
- **Cloudflare Workers Builds（控制台 Git 集成）** 部署到 Worker `yolo-pc`；根目录 `site`、构建 `npm run build`、部署 `npx wrangler deploy`（配置 `site/wrangler.jsonc`）。push main 即自动构建部署。
- 域名走 `SITE_URL` 构建变量（canonical/sitemap 自适应），现为占位 `yolohappy.app`。

## 三、站点结构（路由）
- `/` 首页：楔子 + 三痛点工具 + OTA 对比 + 信任三栏
- `/china-visa-free-checker` ★ 签证检测器（选护照即时裁定 + 完整行程判定）
- `/china-visa-free/[country]` 按国籍静态 SEO 页（~90 有签证关系国 + 16 高流量需签市场）
- `/guides`、`/guides/[city]`、`/guides/[city]/[attraction]` 城市指南（图文/实用信息/子区域，**除音频外全展示**，音频做下载钩子）
- `/attraction-tickets`、`/attraction-tickets/[slug]` 门票（护照姓名校验器 + 放票倒计时 + 按景点 SEO 页）
- `/china-hotels-that-accept-foreigners` 外宾友好酒店**可筛选目录**（城市/星级/价格/接待外国人/英语服务）
- `/about` 信任页；`/404`；`sitemap-index.xml`、`robots.txt`
- 全站常驻 `GetTheApp`

## 四、数据来源与后台（CMS）
构建期从 Supabase 拉取打成快照（`site/src/data/*.json`，均 gitignore），原样渲染：
| 模块 | Supabase 表 | 后台编辑入口（admin-vue） |
|---|---|---|
| 签证 | `visa_policies_v2` / `visa_policy_grants_v2` / `visa_cities` / `visa_city_policy_matrix` / `visa_ports` / `visa_config` | 🛂 签证引擎 v2 |
| 城市指南 | `cities` / `attractions` / `sub_areas`（音频字段构建期剥离） | 城市概览 / 景点 |
| 门票 | `ticket_attractions` | 🌐 官网内容 → 🎟️ 门票 |
| 酒店 | `hotels` | 城市 → 酒店 |
- 签证引擎为 App `VisaPolicyEngine`/`VisaNationalitySummary` 忠实移植成 TS（`site/src/lib/visa-engine.ts`、`visa-summary.ts`）。
- 门票/酒店缺数据时回退到 committed `*.seed.json`，构建不中断。

## 五、CMS 改动自动重建
- SSG 站点改 CMS 不重建不生效 → 做了**自动重建**：Supabase 触发器 `notify_site_rebuild()`（`pg_net` + Vault `cf_deploy_hook_url`）在内容表写入时 POST Cloudflare deploy hook，自动重建 `yolo-pc`（~1-2 分钟）。
- 覆盖表：ticket_attractions / cities / attractions / sub_areas / city_guides / culture_tips / visa_* / **hotels**。
- 迁移：`090`（ticket_attractions 表）· `091`（重建触发器框架 + 站点表）· `092`（hotels 触发器）。Deploy hook URL 存 Vault，**不入公开仓库**。

## 六、安全
- **无匿名/远程可利用漏洞**：无不可信输入到达 HTML 注入点；anon key 只读 + RLS + 不进客户端包；dist 无密钥/音频/完整内容数据泄漏。
- **存储型 XSS**：CMS 富文本经 `sanitize-html` 构建期清洗（仅 build 运行）；客户端 innerHTML 全部 Supabase 字段 `esc()` 转义；酒店富文本备注转纯文本。
- **CSP**：Astro `experimental.csp` 每页 `<meta>` 注入并**自动哈希**内联脚本/样式 → `script-src 'self' 'sha256-…'`（无 unsafe-inline）。
- **响应头**（`_headers`）：X-Frame-Options DENY、nosniff、Referrer-Policy、Permissions-Policy、COOP。

## 七、全设备自适应
- 移动端纯 CSS 汉堡导航（无 JS，CSP 安全）；全局溢出兜底；断点覆盖超小屏→桌面。
- headless Chrome 多宽度实测：mobile(390) `scrollWidth=375` 零溢出；桌面完整导航。
