# 工作日志 - 2026-07-08

## 功能：后台管理员管理功能增强

### 背景
当前后台管理系统已有完善的用户管理功能（UsersHub.vue），但管理员管理功能有限：
- ✅ 可以查看现有管理员列表
- ❌ 无法通过 UI 添加/删除管理员
- ❌ 需要手动在 Supabase SQL 中操作

### 本次改动

#### 1. 数据库迁移 (`supabase/migrations/073_admin_management_policies.sql`)
添加了 `admin_users` 表的 INSERT/DELETE 策略，允许管理员通过 UI 添加/删除其他管理员：
```sql
-- 允许管理员插入新的管理员
CREATE POLICY "Admins insert admin_users" ON admin_users
  FOR INSERT TO authenticated WITH CHECK (public.is_admin());

-- 允许管理员删除管理员
CREATE POLICY "Admins delete admin_users" ON admin_users
  FOR DELETE TO authenticated USING (public.is_admin());
```

#### 2. Edge Function (`supabase/functions/manage-admin/index.ts`)
创建了新的 Edge Function，支持以下操作：
- `create_user` - 创建新用户并添加为管理员（使用 service role key）
- `add_existing_user` - 添加现有用户为管理员
- `remove_admin` - 移除管理员权限
- `list_admins` - 列出所有管理员

安全特性：
- 验证调用者必须是管理员
- 防止自移除管理员权限
- 使用 service role key 进行用户创建

#### 3. 前端工具函数 (`admin-vue/src/lib/supabase.js`)
添加了以下辅助函数：
- `addAdmin(userId, email)` - 添加现有用户为管理员
- `removeAdmin(userId)` - 移除管理员权限
- `createAdminUser(email, password)` - 创建新管理员用户
- `addExistingUserAsAdmin(userId, email)` - 通过 Edge Function 添加管理员
- `isAdmin(userId)` - 检查用户是否为管理员

#### 4. 用户管理 UI 增强 (`admin-vue/src/components/hubs/UsersHub.vue`)

**用户详情页面：**
- 显示当前用户的管理员状态
- "添加为管理员" 按钮（非管理员时显示）
- "移除管理员权限" 按钮（是管理员时显示）

**管理员列表区域：**
- "+ 创建新管理员" 按钮（弹出对话框创建新用户）
- "刷新列表" 按钮
- 每行添加 "移除" 按钮
- 支持从用户详情页直接添加/移除管理员

**创建管理员对话框：**
- 输入邮箱和密码
- 创建新 Supabase 用户并自动添加为管理员

### 部署说明

1. **应用数据库迁移**
   ```bash
   # 在 Supabase Dashboard 的 SQL Editor 中运行
   # 或使用 CLI: supabase db push
   ```

2. **部署 Edge Function**
   ```bash
   cd supabase
   npx supabase functions deploy manage-admin
   ```

   确保在 Supabase Dashboard 中设置了 `SUPABASE_SERVICE_ROLE_KEY` 环境变量。

3. **构建并部署前端**
   ```bash
   cd admin-vue
   npm run build
   # 部署到 Cloudflare
   ```

### 安全考虑

1. Edge Function 验证调用者必须是管理员
2. Service Role Key 仅在 Edge Function 中使用，不暴露到前端
3. 数据库 RLS 策略确保只有管理员可以操作 admin_users 表
4. 防止用户自移除管理员权限
