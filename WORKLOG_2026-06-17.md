# WORKLOG · 2026-06-17

## 客服模块商业化：结束会话 / 历史记录 / 软删除（用户端 + 客服端）

补齐会话生命周期,使 Genius Bar 成为可商用的多用户×多客服系统。决策:**软删除·各删各的**(后台留痕可审计)、**工单模式**(结束=历史只读,再咨询=新会话)、暂保持 3 秒轮询。

### 后端 `068_support_chat_lifecycle.sql`
- `support_conversations` 加 `user_deleted_at` / `agent_deleted_at`(软删标记)+ 用户侧 live 索引。
- 软删/结束都走现有 UPDATE 策略,无需新 RLS;后台(is_admin)读取不过滤 → 全量留痕。

### iOS 用户端
- `SupportConversation` 加 `userDeletedAt/agentDeletedAt` + `isClosed`。
- `SupportChatService`:`loadMyConversations` 改拉 open+closed 且 `user_deleted_at is null`,派生 `activeConversations`/`historyConversations`;新增 `endConversation()`、`deleteMyConversation(_)`;`loadMessages` 加 `.limit(200)`。
- `GeniusBarHomeView`:收件箱分「进行中」「历史会话(只读)」两段,长按 `contextMenu` 删除记录。
- `GeniusBarChatView`:open 时工具栏「结束会话」(确认弹窗);closed 时隐藏输入栏、显示「会话已结束」横幅 +「再次咨询 TA」→ 新建会话。
- `xcodebuild build` EXIT=0。

### web/support 客服端
- 列表查询加 `.is("agent_deleted_at", null)`(排除客服已软删)。
- 查看已结束会话:顶栏「删除记录」(软删 `agent_deleted_at`,后台仍保留)+ 隐藏输入区(只读);未结束仍显示「结束会话」。
- `node --check` 通过。

### admin 后台
- `support_conversations` 只读监控加 `user_deleted_at`/`agent_deleted_at` 字段,运营能看出哪侧删过;admin 读取不受软删过滤(留痕)。

### 验证要点
- 结束 → 进历史只读 → 再咨询新建;用户删历史 → App 消失、客服端/后台仍在;客服删已结束 → 客服端消失、后台仍在;多用户对同一客服 = 多条独立会话。

## 后台：支付建议匹配条件改结构化多选

`payment_advice_rules.match_json` 由「手敲 JSON」改为可视化组合控件 `payment_match`:
- `admin/js/field-types.js`：新增 `renderPaymentMatch`(国家多选[来自 passport_countries] + 卡组织多选[visa/mc/jcb/银联/amex] + 行程下拉) + `readFieldValue` 的 `payment_match` 分支(读回拼成 `{country?,cards_exclude?,trip?}`,不选=`{}`=对所有人显示)。

- 与 iOS `PaymentHelperService` 的匹配逻辑一致(国家大小写不敏感),无需改 App。`node --check` 通过。需 `npm run deploy:admin` 生效。
